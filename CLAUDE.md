## Project overview

A Django project for "Common Ground", a fictional shared counselling practice, built as a portfolio piece. It is currently a minimal single-page marketing site with no custom models, no database-backed content, and no tests yet.

There is no `requirements.txt`/`Pipfile`/`pyproject.toml` in the repo, and Django is **not** installed in the default environment — install it before running anything (`pip install django`; `manage.py` was generated by `django-admin startproject` using Django 6.0.6, so `pip install "django==6.0.6"` matches the generated config most closely).

## Commands

Run all commands from the repo root (where `manage.py` lives):

```bash
python manage.py runserver          # dev server at http://127.0.0.1:8000/
python manage.py migrate            # apply migrations (no custom models yet, so this only sets up auth/admin/sessions tables)
python manage.py makemigrations     # generate migrations after editing core/models.py
python manage.py test               # run the test suite (core/tests.py is currently empty)
python manage.py test core.tests.SomeTestCase.test_method  # run a single test
python manage.py createsuperuser    # create an admin user for /admin/
```

There is no linter or formatter configured in this repo.

## Architecture

Standard two-part Django layout: a project package for site-wide config, and one app for actual functionality.

- `ProjectCounsellingSite/` — the Django project package: `settings.py`, root `urls.py`, `wsgi.py`/`asgi.py`. `ROOT_URLCONF` delegates all non-admin routes to `core.urls`.
- `core/` — the (only) Django app.
  - `core/urls.py` → `core/views.py:index` renders `core/templates/index.html` at `/`. This is currently the entire site.
  - `core/models.py`, `core/admin.py` — empty scaffolding; no models are defined or registered yet.
  - `core/templates/index.html` — all landing-page markup. It links `static/css/index.css` and `static/js/index.js` via `{% static %}` (scroll-reveal via `IntersectionObserver`, animated stat counters, and the hero portrait carousel all live in the JS file). **This is the file to edit for page content/markup changes; edit the static CSS/JS files directly for style/behavior changes.**
- `static/css/index.css`, `static/js/index.js` — all styles and behavior for `index.html`.
- `static/images/face_*.png` — portrait photos used in the hero's `.hero-faces` carousel (`#facesTrack` in `index.html`). The track is duplicated (each image listed twice) for a seamless CSS `translateX` loop; `static/js/index.js` runs a `requestAnimationFrame` loop that scales each photo based on its live distance from the viewport centre (small at the edges, large in the middle) — adding/removing images means updating both copies in the track markup.
- `db.sqlite3` — dev SQLite database (empty; no data model exists yet).

## Settings notes

`ProjectCounsellingSite/settings.py` is the stock `startproject` output and has not been hardened: `DEBUG = True`, a placeholder `SECRET_KEY`, and `ALLOWED_HOSTS = []`. Treat these as dev-only; flag them if asked to prepare this for production.

`STATICFILES_DIRS = [BASE_DIR / 'static']` is required in `settings.py` for `runserver` to discover the project-root `static/` folder — it isn't inside an app, so Django's default app-static autodiscovery won't find it without this.
